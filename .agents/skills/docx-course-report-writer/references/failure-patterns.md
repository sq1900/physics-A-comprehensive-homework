# Failure Patterns To Prevent

These patterns come from recent DOCX report runs. Apply them as checks, not as session stories.

| Symptom | Root Cause | Required Response |
|---|---|---|
| AI-generated concept figure is clean but empty, vague, or stock-like | Prompt lacked a concrete reader question, focal subject, composition, density target, and negative prompt; Critic accepted only "no text/no watermark" | Load `references/ai-image-prompting.md`, write a full prompt card, reject mostly empty/generic outputs, regenerate with narrower text-to-image prompt or replace with a separate non-AI deterministic diagram |
| Reference entry looks plausible but has fake or wrong DOI, author, title, venue, year, pages, URL, or publisher | Citation drafting started before a verified `Reference Metadata Ledger`; the agent trusted memory or generated metadata | Stop drafting, verify DOI/canonical metadata against DOI resolver/Crossref/publisher page/official source, repair `references.md`, and rerun Actor/Critic reference hallucination review |
| Body citations appear as plain `[1]` text instead of superscript bracketed numeric citations | DOCX builder or manual edit did not apply GB/T 7714-2015 sequential citation typography | Regenerate or edit from source so citations are superscript; inspect DOCX/PDF pages, not only extracted text |
| Reference list exists but does not follow GB/T 7714-2015 order, punctuation, type marker, DOI/URL treatment, or reference list typography | "Reference exists" was treated as enough; formatting and metadata audits were skipped | Rebuild from verified ledger using GB/T 7714-2015; check sequence numbers, type markers, DOI/URL fields, font/size, hanging indent, and order by first citation appearance |
| Automatic TOC exists but looks flat, cramped, or visually unclear | QA checked only the DOCX `TOC` field and did not configure or inspect Word `TOC 1/2/3` styles | Configure TOC styles, update fields through Word, render/export the TOC page, and treat "automatic but ugly" as a polish blocker |
| Report delivered but the TOC page says "please update in Word" or the user says the directory/TOC disappeared | DOCX was generated with a TOC field but exported before Word field update, or a later script edit replaced the TOC body | Re-run Word COM field update before PDF export; inspect the TOC page render; add script-level update/export so reruns do not regress |
| TOC page is counted as page 1, or the first body/chapter page starts at page 2/3 | Cover and TOC front matter were separated from the body with only page breaks, or front matter footers still contain a body `PAGE` field | Insert a Word section break after the TOC, remove body page-number fields from front matter footers, restart body page numbering at 1, update fields through Word, and run `qa_docx_report.py --require-body-page-start-1` |
| Chinese filenames become `????`, Word cannot find files, or Python/PDF readers fail on Chinese paths | PowerShell/Python subprocess encoding or Word COM path handling failed | Use UTF-8 output, absolute paths, and an ASCII temp path fallback; avoid passing garbled filenames; verify the final native-path DOCX/PDF exists |
| Template old topic leaks into the new report | Prior report body/media/TOC was not fully removed | Search DOCX text and OOXML for stale terms, old captions, old media count, and old TOC entries before delivery |
| User says a provided template was ignored, or the final report only vaguely resembles the template | The template was treated as a style source or writing reference; the builder created a blank document, cleared the body, or hand-recreated cover/headings/tables | Use `references/template-fidelity.md`: copy the template first, extract a Template Fidelity Contract, replace content inside the copy, preserve sections/styles/header/footer/TOC/media, and compare rendered template/final pages |
| TikZ arrows are misaligned, invisible, crossed, or cramped after insertion | Source diagram looked acceptable before scaling, but final PDF made arrows unreadable | Inspect rendered PNG/PDF and final report page; increase spacing, anchor arrows explicitly, route with orthogonal paths, and regenerate from source |
| Flowchart or architecture diagram looks acceptable at source stage but arrows overlap boxes/text in the final document | No mandatory Review And Revise pass after rendering | Run a dedicated arrow audit: check each arrow start/end anchor, direction, crossing, overlap, arrowhead visibility, and spacing; revise layout until final PDF page is clear and visually balanced |
| TikZ or generated diagram has text touching borders, arrows, or other modules | Node text width, padding, font size, or label wrapping was not reviewed at final scale | Run text-layout audit: labels must stay inside intended areas, avoid all arrows/borders/legends/captions, and remain readable after DOCX/PDF scaling; shorten labels or split the diagram |
| Diagram is still hard to read after several arrow fixes | The layout is too dense; local arrow rerouting cannot fix the global structure | Stop patching the same crowded diagram. Split by phase/lane/chapter or rebuild with a simpler one-direction layout |
| AI image includes unrelated school/company names, fake UI text, wrong numbers, or hallucinated labels | Image was generated before a strict text/spec whitelist, or AI text was trusted | Write per-figure drawing specs first; reject wrong text and regenerate. Do not repair AI images by adding labels after generation; if repeated attempts fail, use a separate non-AI deterministic diagram |
| Result section is too thin or score claims are stale, such as keeping "100%" after the user corrected the final score | Report text, figures, CSV labels, and captions were not tied to a fact ledger | Build result data tables/plots from real files; scan all text and figure sources for forbidden/obsolete values; separate direct model, ensemble, OCR, probe, and final submission claims |
| Official/source screenshot is actually a 403/CAPTCHA/error page | Browser capture was accepted without semantic inspection | Inspect screenshot content; replace with official metadata/source cards when live page access is blocked |
| Final answer says QA passed while PDF/page render was unavailable | DOCX text QA was confused with visual QA | State the limitation explicitly, or complete PDF/rendered-page QA before claiming layout is verified |
| Rendered pages were checked before Word updated the DOCX fields | The agent rendered a stale DOCX/PDF and missed TOC/page-number defects | Word COM update/export must happen before PDF page rendering; then render the final PDF pages to PNG with `scripts/render_pdf_review_pages.py`, inspect individual pages or four pages per contact sheet, and label these as PDF page render images, not screenshots |
| Four-page contact sheet contains an unnoticed blank quadrant | The agent verified the contact sheet was non-empty overall instead of checking each page cell; the render script lacked blank-page detection | Run `scripts/render_pdf_review_pages.py` with default blank-page detection, treat `BLANK_PAGE` as blocking, inspect red-bordered near-blank pages individually, and document any intentional blank page before using `--allow-blank-pages` |
| User reports a defect after delivery and the next fix edits only the final DOCX | Source-first workflow was abandoned after feedback | Treat feedback as a failed QA test; fix the source script/draft/figure spec, regenerate DOCX/PDF, and rerun focused checks |
| Repair starts with a guessed fix instead of inspecting the actual failed artifact | Systematic debugging was skipped under pressure | Invoke `superpowers:systematic-debugging` when installed; otherwise inspect artifact/source/logs, compare with working patterns, form one hypothesis, make one source-level fix, and verify |
| No-template report starts with a blank page before the TOC | Report builder inserted a pre-TOC page break even though no cover/template opening was preserved | Only add a page break before TOC when a real cover/opening exists; render page 1 after PDF export and reject accidental blank first pages |
| Contact sheet looked acceptable but an individual page was blank or near-blank | Critic used the contact sheet as proof and did not inspect every rendered page PNG | Render every PDF page, run blank/near-blank detection, inspect individual page PNGs, and treat contact sheets as an index only |
| Default template brings old static TOC, placeholders, or sample chapter text into a new report | Integrated template was loaded as full body content instead of a style/page setup source | Use the user-provided template when available; otherwise use `skill-assets/default-course-report-template.docx` but clear stale body content unless cover/body preservation is explicitly requested |
