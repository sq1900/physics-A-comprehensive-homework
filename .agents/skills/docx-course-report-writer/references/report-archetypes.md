# Report Archetypes

Select the report type early because it determines evidence, structure, and QA priorities.

## Experiment Report

Prioritize:

- assignment requirement mapping
- environment and tool versions
- runnable implementation or procedure
- real result evidence
- screenshots/logs/plots from actual runs
- method and code explanation
- test matrix
- abnormal/error handling
- problems and solutions
- experiment summary and learning

Useful tables:

- requirement-to-implementation mapping
- environment table
- file responsibility table
- protocol/API table
- normal test table
- abnormal test table

Useful figures:

- real terminal/result screenshots
- workflow or protocol flowchart
- architecture/process diagram
- plotted local result data

## Course Paper

Prioritize:

- clear thesis
- literature-backed argument
- method or concept explanation
- comparison dimensions
- synthesis rather than source-by-source summary
- citations at point of use

Useful visuals:

- conceptual framework diagram
- comparison table
- method taxonomy
- paper figure crop when directly analyzed

## Literature Review

Prioritize:

- source quality
- topic taxonomy
- chronological or method-based organization
- comparison and gap analysis
- recent authoritative sources when the topic is time-sensitive

Useful visuals:

- research map
- method comparison table
- timeline
- paper-crop figures with attribution

## Reading Report

Prioritize:

- accurate summary of the assigned material
- personal understanding and critique
- key concept explanation
- limited but meaningful evidence

Useful visuals:

- structure map
- concept relation diagram
- small quotation/evidence table when required

## Course Summary

Prioritize:

- course topic structure
- knowledge map
- selected examples
- learning reflection
- concise synthesis rather than exhaustive notes

Useful visuals:

- chapter map
- concept hierarchy
- comparison table
